🌿 HEALONE – FINAL PROJECT STRUCTURE
🎯 Project Title

HealOne – Ayurvedic Wellness Ecosystem Platform

🏗️ SYSTEM OVERVIEW

HealOne is a role-based MERN stack platform that combines:

Ayurvedic Product E-Commerce

Natural Remedies (Gharelu Nuskha)

Yoga Guidance

Wellness Programs (Service-Based)

Expert-Verified Content System

It is NOT a doctor-patient platform.

It is a wellness ecosystem.

👨‍⚕️ 2️⃣ SPECIALIST PANEL (Content Creator)
Role:

Ayurvedic wellness contributor.

Cannot control system.

Specialist Can:

✔ Add Remedy
✔ Add Yoga
✔ Create Wellness Program
✔ Write educational content
✔ View approval status

Cannot:
❌ Delete other content
❌ Manage users
❌ Access analytics

Content goes to:

Pending → Admin Approval → Published

This shows moderation workflow.

👤 3️⃣ USER PANEL (Registered User)

✔ Enroll in programs

🌍 4️⃣ GUEST PANEL (Public View)

No login required.

✔ Read content

Cannot:
❌ Buy product
❌ Enroll in service
❌ Save data

🧠 HOW EVERYTHING CONNECTS (Core Logic)
Connection Through: Health Category

Example:

Health Category = “Stress”

Connected:

Stress Relief Oil (Product)

Stress Remedy (Remedy)

Stress Yoga Routine (Yoga)

7-Day Stress Relief Program (Service)

When user selects “Stress”:

System shows all related content.

That is your smart ecosystem logic.
